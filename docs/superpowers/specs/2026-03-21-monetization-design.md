# 健常者エミュレータ事例集 収益化設計

## 背景と目的

健常者エミュレータ事例集は、社会生活の暗黙知を言語化・集積するUGCプラットフォームである。11,253件の投稿、517人の登録ユーザー、月間約195K PV（2026年3月時点、急成長中）の規模を持つ。

現在の収益はゼロ。過去にGoogle AdSenseを導入したが、収益が低くUXを損なう広告が多かったため撤退。投げ銭機能は存在するが機能していない。

**目標**: 月額30万円以上の持続可能な収益構造を構築する。

## 設計原則

1. **投稿の閲覧は無料を維持する** — サイトのミッション（知識格差の解消）と整合させる
2. **広告モデルは採用しない** — UXを損なう
3. **収益化は付加価値で行う** — 「読む」は無料、「より深く使う」「商用利用する」は有料
4. **自動化優先** — 運営の手間は最小限に。週数時間のコンテンツ制作は許容

## フェーズ1: プレミアムサブスクリプション

### 概要

ロイヤルユーザー向けに、投稿体験を強化する付加価値機能を月額課金で提供する。

### 価格

- **月額580円**（税込）
- 単一プラン、年額プランなし

### プレミアム機能

#### 1. AI事例パーソナライズ検索

ユーザーが自然文で状況を入力すると、11,253件の投稿からAIが最も関連する事例を検索・要約して返す。

- **技術基盤**: 既存のCloudflare Vectorize（類似投稿検索）+ LLM層を追加
- **無料ユーザー**: 月3回まで
- **プレミアムユーザー**: 無制限

#### 2. 投稿者ダッシュボード

自分の投稿がどれだけ人の役に立っているかを可視化する「承認体験」ツール。

- 概要パネル: 累計投稿数、累計いいね数、累計閲覧数
- 投稿ごとのパフォーマンス一覧（PV、いいね、コメント数）
- ハイライト: 「あなたの最も多くの人を助けた投稿」TOP3、「今週最もいいねが増えた投稿」、ブックマークされた数
- 月別トレンドグラフ

**データソース**:
- PV: BigQuery（GA4）
- いいね/コメント/ブックマーク: D1データベース

**前提条件**: 現在、投稿はIPハッシュで匿名化されておりユーザーIDと紐付いていない。ログインユーザーと投稿の紐付け機能を新規実装する必要がある。

#### 3. プレミアムバッジ

プレミアム会員の投稿・コメントにバッジを表示する。コミュニティでの可視性を高め、投稿者としてのアイデンティティを強化する。

### 決済基盤

- Stripe Subscriptions（月額定期課金）
- Stripe Customer Portal（プラン管理・解約セルフサーブ）
- Webhook でサブスクリプション状態をD1に同期

### 収益目標

月額580円 × 300〜600人 = **月15〜35万円**

CVR目標: ロイヤルユーザーの2〜4%（844人ベース）。トラフィック成長に伴いベースも拡大。

## フェーズ2: コンテンツライセンス販売

### 概要

投稿データを元にテーマ別キュレーション記事を運営側で制作し、全文無料公開する。記事を研修・教育目的で商用利用するためのライセンスをStripeで販売する。

**「読む」は無料、「使う」は有料**というモデル。

### テーマ別記事

#### コンテンツ構造

- 導入: テーマの概要、なぜこの暗黙知が重要か
- 事例: 投稿データから厳選した事例（要約+解説付き）
- 実践アドバイス: 事例から得られる具体的な行動指針
- 全文をSSRで公開（SEOクローラブル）

#### テーマ例

- 「職場で空気が読めないと言われたときの対処事例集」
- 「飲み会・会食のマナーで失敗しないための暗黙知」
- 「Discord・SNSでの暗黙のルール集」
- 「就活・面接で知っておくべき暗黙の了解」

#### 制作フロー

1. BigQuery + D1のデータから、いいね数の多い投稿をテーマ別に抽出
2. 運営が週数時間かけてキュレーション+解説を執筆
3. サイト上に公開
4. 月1〜2本のペースで積み上げ

### ライセンス販売

#### 想定顧客

- 企業の人事・研修部門（コミュニケーション研修、新人研修）
- 就労支援事業所・福祉施設
- 学校・教育機関
- キャリアコンサルタント、カウンセラー

#### ライセンスで許諾される権利

- 社内研修・教育目的での複製・配布
- スライド・教材への引用・転載
- PDF化して社内共有
- **許諾されないもの**: 再販、外部公開、改変後の再配布

#### 価格体系

| プラン | 内容 | 価格 |
|---|---|---|
| 単品 | 1記事のライセンス | 5,000円 |
| テーマパック | 1テーマの全記事 | 30,000円 |
| 年間フリー | 全記事使い放題（1年） | 月額15,000円 |

#### ユーザー導線

```
Google検索「職場 暗黙のルール」
  → テーマ別記事ページ（全文無料で読める）
  → 記事下部にCTA:「この記事を研修・教育で使いませんか？」
  → ライセンス購入ページ（Stripe Checkout）
  → 決済完了 → ライセンス証書PDF + 利用規約を自動メール送付
```

#### 技術的な実装

- Stripe Checkout Session で決済
- 決済完了webhook → ライセンス証書PDF自動生成 → メール送付
- ライセンス管理テーブル（組織名、メール、対象記事、有効期限）
- 記事ページ自体にはアクセス制限なし

### SEO効果

- 全文無料公開のため、一般キーワード（「職場 暗黙のルール」等）での検索流入を獲得
- 記事からサイト本体への内部リンクで投稿への導線も構築
- 積み上げ型でロングテール流入が増加

### 収益目標

年間ライセンス10〜20社 + 単品・パック購入 = **月15〜45万円**

## 技術的な前提条件・新規実装が必要なもの

1. **Stripe連携基盤**: Checkout Session、Subscription、Webhook、Customer Portal
2. **ユーザーと投稿の紐付け**: ログインユーザーのUUIDを投稿に関連付ける仕組み
3. **プレミアム状態管理**: D1にサブスクリプション状態テーブル、ミドルウェアでプレミアム判定
4. **AI事例検索**: Vectorize + LLM（Cloudflare Workers AI or 外部API）による自然文検索・要約
5. **投稿者ダッシュボード**: BigQuery PVデータの取得API、D1データの集計、フロントエンドグラフ
6. **ライセンス管理**: ライセンステーブル、PDF生成、メール送信
7. **テーマ別記事ページ**: 新規ルート、記事コンテンツ管理（D1 or Markdown）
