# E2E テストシナリオ定義

## シナリオ 1: トップページ

**URL**: `http://localhost:3000/`

確認事項:
- タイトルに「トップページ」が含まれる
- 以下のセクションが全て表示される
  - `.latest-posts` — 最新投稿
  - `.voted-posts` — 最近いいね
  - `.recent-comments` — コメント
  - `.community-posts` — コミュニティ
  - `.famed-posts` — 殿堂入り

---

## シナリオ 2: フィードページ

**URL**: `/feed` （トップページの「最新の投稿を見る」から遷移）

確認事項:
- 「最新の投稿を見る」リンクをクリックするとフィードページへ遷移し、タイトルに「フィード」が含まれる
- `.feed-posts section` に投稿一覧が表示される
- ページネーション（`›` / `‹` ボタン）が表示される
- `.feed-type-select select` でタブ切替ができる
  - `likes` → タイトルに「いいね」
  - `unboundedLikes` → タイトルに「無期限」
  - `timeDesc` → タイトルに「フィード」

---

## シナリオ 3: ランダムページ

**URL**: `/?tab=random`（`/random` は `/?tab=random` にリダイレクト）

確認事項:
- タイトルは「トップページ」（トップページのランダムタブとして実装）
- `.random-posts` セクションが表示される
- `.recent-comments` セクションが表示される

---

## シナリオ 4: サイト説明ページ

**URL**: `/readme`

確認事項:
- タイトルに「サイト説明」が含まれる

## シナリオ 5: 検索ページ（DuckDB-WASM）

**URL**: `/search`

> **前提条件**: GCS 環境変数（`SEARCH_PARQUET_FILE_NAME` / `TAGS_PARQUET_FILE_NAME` / `GOOGLE_CLOUD_PROJECT_ID` 等）が設定されていること。
> 未設定の場合、loader が 500 エラーを返してページが表示されない → **SKIP** 扱い。

確認事項:
- タイトルに「検索」が含まれる
- `.search-input-form` が表示されるまで待機（DuckDB-WASM 初期化、最大 30 秒）
- キーワード「Twitter」を入力 → `.search-results .post-title` に結果が表示される
- タグ選択 Accordion を開き「やってはいけないこと」を選択 → 結果が表示される

---

## シナリオ 6: 投稿フォーム

**URL**: `/post`

確認事項:
- タイトルに「投稿」が含まれる
- フォームを上から順に入力して「投稿する」を押すとプレビューモーダルが開く

入力手順（上から順に）:
1. カテゴリ: デフォルト（misDeed）のまま
2. 5W1H+Then（各フィールドの id セレクター）:
   - `#who` ← 「テストユーザーが」
   - `#what` ← 「友人に」
   - `#when` ← 「昨日」
   - `#where` ← 「公園で」
   - `#why` ← 「面白そうだったから」
   - `#how` ← 「冗談を言った」
   - `#then` ← 「空気が悪くなった」
3. 健常行動ブレイクポイント: placeholder `"友人の言動は冗談だという事に気が付く必要があった"` のフィールド
4. どうすればよかったか: placeholder `"冗談に対してただ笑うべきだった"` のフィールド
5. タイトル: placeholder `"タイトル"` のフィールド
6. 「投稿する」ボタンをクリック
7. `dialog.modal-open`（プレビューモーダル）が表示されることを確認

バリデーションエラーは `[role='status']` のトースト通知に表示される。
クリア操作: 「入力内容をリセット」→ モーダルで「リセットする」の 2 ステップ。
