## シナリオ P1: 初期表示

- `/post` を開く
- 「投稿タイプを選択」が表示されることを確認する
- 初期カテゴリが `結果悪` であることを確認する
- 5W1H+Then の各入力欄が表示されることを確認する
- Turnstile と「投稿する」ボタンが表示されることを確認する
- 最低1つの textarea に文字列を入力できることを確認する
- タイトル入力欄には `プログラムテスト` を使用する

## シナリオ P2: カテゴリ切替

- `結果善` を選ぶ
- 見出しが「なぜやってよかったのか」「やらなかったらどうなっていたか」に切り替わることを確認する
- `知識募集` を選ぶ
- 見出しが「試したこと」「まだやってないこと」に切り替わることを確認する
- `結果悪` に戻す
- 見出しが元に戻ることを確認する

## シナリオ P3: 動的入力とタグUI

- 「書ききれなかった前提条件はありますか？」の `追加` を押す
- textarea が増えることを確認する
- `削除` で消えることを確認する
- タグ選択、タグ作成、タグプレビューが連動することを確認する

## シナリオ P4: バリデーション

- 必須項目未入力のまま first submit 相当の操作を試し、toast エラーが出ることを確認する
- その後、必須項目の textarea に最低限の文字列を入力し、タイトルには `プログラムテスト` を入れて、空入力だけが原因のバリデーションではなくなることを確認する
- Turnstile 完了前は送信ボタンが有効化されないことを確認する

## シナリオ P5: localStorage 保存

- いくつか入力する
- ページを再読み込みする
- 入力内容が残ることを確認する
- クリア操作後に再読み込みし、内容が消えていることを確認する

## シナリオ P6: プレビュー

- 必須項目を埋めてプレビュー取得まで進める
- モーダルが開くことを確認する
- タイトルと HTML プレビューが表示されることを確認する
- 「修正する」でモーダルが閉じることを確認する
- コピー操作が失敗しないことを確認する

## シナリオ P7: 投稿→コメント→編集→削除の一連テスト

**このシナリオはE2Eテスト時に通しで実行する。投稿は必ず `プログラムテスト` をタイトルに含めること。**

### Step 1: 投稿

- `/post` で `結果悪` カテゴリを選択する
- タイトルに `プログラムテスト投稿` を入力する
- 5W1H+Then の各項目に適当なテストデータを入力する
- Turnstile 認証を完了する
- 「投稿する」を押して second submit を実行する
- 成功 toast が表示されることを確認する
- `/archives/:postId` に遷移することを確認する
- 遷移先で投稿内容が正しく表示されていることを確認する
- **postId をメモする（後続ステップで使う）**

### Step 2: コメント

- Step 1 で遷移した記事詳細ページで、コメント入力欄を探す
- コメント名を `テスター` に変更する
- コメント内容に `E2Eテストコメント` と入力する
- コメントを送信する
- コメントが記事詳細ページに表示されることを確認する

### Step 3: 編集

- 記事詳細ページの「編集する」ボタンを押す
- ログイン済みであれば編集ページに遷移することを確認する
- タイトルを `プログラムテスト投稿（編集済み）` に変更する
- 「更新する」を押す
- 成功 toast が表示されることを確認する
- 記事詳細に戻り、タイトルが更新されていることを確認する

### Step 4: 削除

- テストで作成した投稿を D1 から直接削除する
- 削除コマンド: `npx wrangler d1 execute healthy-person-emulator-db --remote --command="PRAGMA foreign_keys = ON; DELETE FROM dim_posts WHERE post_id = {postId};"`
- 削除後、`/archives/:postId` にアクセスして 404 になることを確認する
