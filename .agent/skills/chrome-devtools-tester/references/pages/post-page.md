# Post Page Scenarios

対象実装:
- [_layout.post.tsx](/Users/sorachi/code/healthy-person-emulator-dotorg/app/routes/_layout.post.tsx)

## シナリオ P1: 初期表示

- `/post` を開く
- 「投稿タイプを選択」が表示されることを確認する
- 初期カテゴリが `結果悪` であることを確認する
- 5W1H+Then の各入力欄が表示されることを確認する
- Turnstile と「投稿する」ボタンが表示されることを確認する

## シナリオ P2: カテゴリ切替

- `結果善` を選ぶ
- 見出しが「なぜやってよかったのか」「やらなかったらどうなっていたか」に切り替わることを確認する
- `知識募集` を選ぶ
- 見出しが「試したこと」「まだやってないこと」に切り替わることを確認する
- `結果悪` に戻す
- 見出しが元に戻ることを確認する

## シナリオ P3: 動的入力とタグUI

- 「書ききれなかった前提条件はありますか？」の `追加` を押す
- textarea が増えることを確認する
- `削除` で消えることを確認する
- タグ選択、タグ作成、タグプレビューが連動することを確認する

## シナリオ P4: バリデーション

- 必須項目未入力のまま first submit 相当の操作を試し、toast エラーが出ることを確認する
- Turnstile 完了前は送信ボタンが有効化されないことを確認する

## シナリオ P5: localStorage 保存

- いくつか入力する
- ページを再読み込みする
- 入力内容が残ることを確認する
- クリア操作後に再読み込みし、内容が消えていることを確認する

## シナリオ P6: プレビュー

- 必須項目を埋めてプレビュー取得まで進める
- モーダルが開くことを確認する
- タイトルと HTML プレビューが表示されることを確認する
- 「修正する」でモーダルが閉じることを確認する
- コピー操作が失敗しないことを確認する

## シナリオ P7: 投稿完了

- 実行コストが高いので、投稿機能そのものを触った変更時だけ行う
- second submit 後、成功toast が表示されることを確認する
- 約2秒後に `/archives/:postId` に遷移することを確認する
